# FYP-stream-processing

The main idea is to build an interactive tool that tracks and visualizes data flow in a simple analytics pipeline. For this, we will use a social media scenario: tweets flow into the system, hashtags are extracted and counted, and trending scores are computed. When a hashtag trends, users often want to know: Which tweets caused this? From which users? This is the core idea behind data provenance; tracking the origins and history of data as it moves through a system. Your tool will record which tweets contributed to each hashtag's score and display this information visually. The tool will be able to add logging to track contributions, store the records in a simple format (e.g., JSON or SQLite), and build a web-based dashboard where users can click on a hashtag to see which tweets contributed to it. The focus is on creating a clear, interactive visualization rather than optimizing for performance. Lastly, we will design the interface to effectively communicate data lineage and present your demo with sample data that illustrates the concept.
